8. Storyboard (Detailed)

The storyboard below explains the flow of the 2D web game Hostage & Rescue from the first screen until the level is completed. Each scene includes the visual focus, user action, system response, and multimedia elements needed for development.

No. Scene / Screen Visual Description Player Action System Response Audio / UI Notes
1 Opening / Splash Screen The game logo appears with a dark security-room background, blinking CCTV light, and pixel-style title text. Player waits briefly or clicks anywhere to continue. The game loads assets and moves to the main menu after a short transition. Short suspense intro sound; fade-in animation for logo.
2 Main Menu Main menu shows the title Hostage & Rescue, with Play, Level Select, Settings, and Credits buttons. Player clicks Play to start or chooses another menu option. Play opens the first available level. Level Select opens the level list. Settings opens audio controls. Button hover effect; simple click sound; background loop begins.
3 Level Select A level map or list displays unlocked and locked levels. Locked stages have a lock icon. Player selects an unlocked level. The selected level loads. Locked levels show a message that the level must be unlocked first. Small warning sound for locked levels; transition to loading screen.
4 Mission Briefing A short briefing panel explains the mission: infiltrate the area, avoid guards, collect evidence, rescue the hostage, and exit safely. Player reads the mission and clicks Start Mission. The game starts the selected level and places the player at the entry point. Typing or panel slide animation; tense background music.
5 Gameplay - Infiltration Player character enters the facility. Guards patrol the area, CCTV cameras rotate, and doors block some paths. Player moves with keyboard controls and observes guard patterns. The player can explore the map. If detected, the warning state is triggered. Footstep sound, guard alert cue, camera scanning animation.
6 Gameplay - Interaction Interactive objects such as computers, keycards, evidence items, and doors are highlighted when the player is close. Player presses the interaction key to collect evidence, open doors, or hack computers. Collected items are added to inventory. Hacked computers can disable CCTV or unlock a route. Interaction icon appears; short pickup or hacking sound.
7 Danger / Detection State If the player enters a guard or CCTV vision area, the screen shows an alert effect and the guard begins to chase. Player must hide, move away, or restart if caught. A warning timer appears. If the player escapes, the alert returns to normal; if caught, the level fails. Alarm sound, red screen border, faster music during chase.
8 Hostage Rescue The hostage is located in a locked room. The room contains a rescue point or interactable hostage sprite. Player reaches the hostage and presses the interaction key. The hostage is rescued and follows the player or is marked as rescued. The exit objective becomes active. Rescue confirmation sound; objective checklist updates.
9 Exit Objective After rescuing the hostage and collecting required evidence, the exit door is highlighted. Player guides the character to the exit door. If all objectives are complete, the level ends. If not, the game shows which objective is missing. Door unlock sound; objective completion animation.
10 Level Complete A result screen shows score, completion time, evidence collected, detections, and stars or rank. Player clicks Next Level, Replay, or Main Menu. Next Level unlocks the following stage. Replay restarts the same level. Victory sound; score count-up animation.
11 Game Over A failed mission screen appears when the player is caught or fails an objective. Player clicks Retry or Main Menu. Retry reloads the level. Main Menu returns to the menu screen. Low suspense sound; retry button highlighted.
12 Settings / Credits Settings screen contains volume controls. Credits screen lists team members, tools, and asset sources. Player adjusts audio or returns to the menu. Settings are saved for the current session. Credits can be opened from the main menu. Slider movement sound; clean pixel UI layout.

Budget note: The estimated total cost remains Rp920,000. The cost can be reduced if the team creates its own pixel assets, sound effects, or background music, or uses free licensed resources.
